【工程实践】如何开始我的个人AI终端 (Streamlit + Gemini)

Posted on 2025-12-30 Edited on 2026-04-07 In Engineering

为了在本地实现一个可控、可记忆、且支持多模型切换的 AI 交互环境，我编写了一个基于 Streamlit 的 Python 脚本 (ai_lab.py)。本文记录了该系统的启动流程与核心配置。

1. 环境准备 (Prerequisites)

在启动前，请确保已安装以下依赖库（只需安装一次）：

  • Python Version: 3.8+ (推荐 3.12)
  • Network: 必须开启系统代理（TUN模式），确保终端能访问 Google API。

3. 使用指南 (Usage Guide)

A. 配置密钥 (Configuration)

  1. 在左侧 Sidebar (侧边栏) 中，找到 Gemini API Key 输入框。
  2. 填入你的 Google AI Studio 密钥 (AIza...)。
  3. 模型选择：
    • gemini-2.5-flash：主力模型，速度快，配额足（推荐）。
    • gemini-3.0-pro-preview：实验性模型，逻辑最强，但每日配额极少（慎用）。

B. 智能功能 (Agent Features)

  • 联网搜索：在提问中包含 “搜索”、“查一下” 等关键词，系统会自动调用 Google Search API 并注入上下文。
  • 时间感知：系统会自动获取当前北京时间，AI 知道今天是几号。
  • 持久化记忆：对话记录会自动保存到本地 chat_memory.json，重启不丢失。

C. 快捷操作

  • 清空历史：点击侧边栏的 🗑️ Clear History 按钮，重置对话并删除本地记录。
  • 热重载：修改 ai_lab.py 代码后，点击网页右上角的 Rerun 即可实时生效。

4. 常见问题 (Troubleshooting)

  • Q: 报错 ConnectionError？
    • A: 检查梯子是否开启。Python 脚本不走浏览器插件代理，必须开系统级代理。
  • Q: 报错 Quota Exceeded？
    • A: 切换回 Flash 模型。Pro 模型配额耗尽后需等待 24 小时。
  • Q: 找不到 streamlit 命令？
    • A: 坚持使用 python -m streamlit run ... 的方式运行。
